# 옵시디언 볼트 정리 가이드

## 1. 폴더 구조 규칙

```
📁 볼트 루트/
├── 📁 Inbox/              ← 새 메모는 무조건 여기에 먼저 저장
│   └── 📁 Files/          ← Inbox 첨부파일
├── 📁 Projects/           ← 진행 중인 프로젝트별 폴더
│   ├── 📁 기어박스_설계/
│   ├── 📁 회전익기_동력전달/
│   └── 📁 투자_재무설계/
├── 📁 References/         ← 참고자료 (논문 번역, 기술 동향 등)
├── 📁 Seminars/           ← 세미나/교육 관련 자료
│   ├── 📁 회차별자료/
│   ├── 📁 Homework/
│   └── 📁 공지사항/
├── 📁 Daily Notes/        ← 일일 메모 (날짜별 자동 생성)
├── 📁 Templates/          ← 템플릿 모음
├── 📁 Guides/             ← 가이드 문서 모음
├── 📁 Attachments/        ← 이미지, PDF 등 첨부파일 통합 관리
└── 📁 Archive/            ← 완료된 프로젝트, 오래된 자료
```

### 각 폴더의 역할

| 폴더 | 역할 | 예시 |
|------|------|------|
| **Inbox** | 아직 분류하지 않은 새 메모 | 검색 결과, 임시 메모, 테스트 |
| **Projects** | 현재 진행 중인 프로젝트 | 기어박스 설계, 하모닉 드라이브 |
| **References** | 참고 문헌, 번역, 기술 동향 | 논문 번역본, 기술 보고서 |
| **Seminars** | 세미나/교육 자료 | 회차별 학습자료, 과제 |
| **Daily Notes** | 날짜별 일일 기록 | 2026-02-19.md |
| **Templates** | 재사용 가능한 문서 템플릿 | 프로젝트 템플릿, 회의록 템플릿 |
| **Guides** | 도구 사용법, 워크플로우 가이드 | 이 문서, KISSsoft 가이드 |
| **Attachments** | 모든 이미지/파일 첨부물 | PNG, PDF, Canvas 파일 |
| **Archive** | 완료되었거나 더 이상 활성이 아닌 자료 | 지난 프로젝트 |

## 2. 파일 명명 규칙

### 기본 원칙
- **한글 사용 가능**: `기어박스_설계_AGMA.md` (O)
- **공백 대신 언더스코어**: `기어박스_설계.md` (O), `기어박스 설계.md` (X)
- **날짜 포함 시 ISO 형식**: `2026-02-19_회의록.md`
- **카테고리 접두사 사용**:
  - `번역_` : 논문/문서 번역본
  - `요약_` : 요약 문서
  - `가이드_` : 사용법/가이드
  - `보고서_` : 분석/동향 보고서

### 예시
```
번역_슈라우딩의_기어_윈디지_손실_저감_효과.md
요약_회전익기_동력전달장치_개발동향.md
가이드_KISSsoft_스크립트_자동화.md
2026-02-19_일일메모.md
```

## 3. 프론트매터(Front Matter) 활용

모든 MD 파일 상단에 YAML 프론트매터를 추가합니다:

```yaml
---
created: 2026-02-19
tags: [기어박스, 설계, AGMA]
category: 프로젝트
project: 기어박스_설계
status: 진행중    # 진행중 / 완료 / 보류
---
```

### 주요 필드 설명

| 필드 | 설명 | 값 예시 |
|------|------|---------|
| `created` | 생성 날짜 | `2026-02-19` |
| `tags` | 검색용 태그 (복수) | `[기어박스, AGMA, 설계]` |
| `category` | 문서 종류 | `프로젝트`, `참고자료`, `가이드`, `세미나` |
| `project` | 소속 프로젝트명 | `기어박스_설계` |
| `status` | 문서/프로젝트 상태 | `진행중`, `완료`, `보류` |

## 4. Claude Code와 함께 쓰는 워크플로우

### 일상적인 작업 흐름

```
1. 새 메모/파일 → Inbox/ 에 저장
2. 주기적으로 (매일 또는 주 1회) → 정리 에이전트 실행
3. 에이전트가 Inbox 파일을 분석 → 적절한 폴더로 이동
4. 프론트매터 자동 추가/수정
```

### 정리 에이전트 실행 방법

터미널에서 다음 명령어로 실행:
```bash
claude --agent-prompt vault-organizer
```

또는 Claude Code 대화 중:
```
/vault-organizer 실행해줘
```

## 5. 옵시디언 추천 설정

### 핵심 플러그인
- **Templater**: 템플릿 자동 적용
- **Calendar**: Daily Notes와 연동
- **Dataview**: 프론트매터 기반 동적 목록 생성
- **Tag Wrangler**: 태그 일괄 관리

### 옵시디언 설정 권장사항
1. **새 파일 생성 위치**: `Inbox` 폴더로 설정
   - 설정 → 파일 및 링크 → 새로 만든 노트를 저장할 위치 → `지정된 폴더` → `Inbox`
2. **첨부 파일 위치**: `Attachments` 폴더로 설정
   - 설정 → 파일 및 링크 → 첨부 파일 기본 위치 → `아래에 지정된 폴더` → `Attachments`
3. **Daily Notes 위치**: `Daily Notes` 폴더로 설정

## 6. 정리 체크리스트

주기적으로 확인할 항목:

- [ ] Inbox에 미분류 파일이 남아 있는가?
- [ ] 프론트매터가 없는 파일이 있는가?
- [ ] 루트 폴더에 직접 놓인 파일이 있는가? (가이드 제외)
- [ ] 불필요한 테스트/임시 파일이 있는가?
- [ ] 첨부파일이 Attachments 외부에 있는가?
